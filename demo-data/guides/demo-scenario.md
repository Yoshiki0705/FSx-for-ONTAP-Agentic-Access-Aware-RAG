# デモシナリオガイド

## 概要

Permission-aware RAGシステムのデモ手順です。2種類のユーザーでログインし、同じ質問に対して異なる検索結果が返ることを確認します。

## デモユーザー

| ユーザー | メール | パスワード | 権限レベル | アクセス可能ドキュメント |
|---------|--------|-----------|-----------|----------------------|
| 管理者 | admin@example.com | DemoAdmin123! | 全ドキュメント | public, confidential, restricted |
| 一般ユーザー | user@example.com | DemoUser123! | 公開のみ | public |

## デモシナリオ

### シナリオ1: 管理者ユーザーでの検索

1. CloudFront URLにアクセス
2. `admin@example.com` / `DemoAdmin123!` でサインイン
3. チャットで以下の質問を入力:

**質問例:**
- 「会社の売上はいくらですか？」
  - 期待結果: 財務レポートの情報を含む回答（¥500,000,000等）
  - ソース: `confidential/financial-report.md`

- 「リモートワークのポリシーを教えてください」
  - 期待結果: 人事ポリシーの詳細を含む回答
  - ソース: `confidential/hr-policy.md`

- 「プロジェクトのスケジュールは？」
  - 期待結果: プロジェクト計画の情報を含む回答
  - ソース: `restricted/project-plan.md`

- 「製品の概要を教えてください」
  - 期待結果: 製品カタログの情報を含む回答
  - ソース: `public/product-catalog.md`

### シナリオ2: 一般ユーザーでの検索

1. サインアウト
2. `user@example.com` / `DemoUser123!` でサインイン
3. 同じ質問を入力:

**質問例:**
- 「会社の売上はいくらですか？」
  - 期待結果: 財務情報にアクセスできないため、公開情報のみで回答
  - ソース: `public/company-overview.md`（財務レポートは非表示）

- 「リモートワークのポリシーを教えてください」
  - 期待結果: 人事ポリシーにアクセスできないため、情報なしまたは公開情報のみ

- 「製品の概要を教えてください」
  - 期待結果: 製品カタログの情報を含む回答（公開ドキュメントなのでアクセス可能）
  - ソース: `public/product-catalog.md`

## Permission-aware動作の確認ポイント

1. **同じ質問、異なる結果**: 管理者と一般ユーザーで検索結果が異なることを確認
2. **ソースドキュメント表示**: 回答に使用されたドキュメントのcitation情報を確認
3. **機密情報の保護**: 一般ユーザーにconfidential/restrictedドキュメントの内容が表示されないことを確認
4. **公開情報のアクセス**: 両ユーザーともpublicドキュメントにアクセスできることを確認

## トラブルシューティング

- **検索結果が返らない**: Bedrock KBデータソースの同期が完了しているか確認
- **権限フィルタリングが効かない**: DynamoDBキャッシュテーブルのTTL設定を確認
- **サインインできない**: Cognito User Poolのユーザーステータスを確認
